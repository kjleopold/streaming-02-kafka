## Custom Project

### Dataset

The dataset used by the Kafka producer was sales.csv. The dataset contains
simulated sales data representing online course purchases from different regions
and customers.

Each record includes fields such as:

- order_id
- datetime
- region_id
- currency_code
- product_id
- unit_price
- quantity
- is_online
- customer_id
- is_new_customer
- device_type
- payment_method
- referral_source
- discount_code
- customer_note

I used the original sales dataset provided in the project, but modified the
streaming messages by enriching them with additional calculated fields before
sending them through Kafka.

### Kafka Messages

The Kafka producer sent enriched sales transaction messages through Kafka one message at a time.

The Kafka topic used was streaming-02-kafka-kjleopold.

The message key used was the region_id field. This allowed messages to be
grouped by geographic region within Kafka partitions.

The producer modified the original message data by adding calculated business
intelligence fields, including: total_revenue and order_priority

The total_revenue field was calculated using the unit price multiplied by the
quantity purchased. The order_priority field classified transactions as LOW,
MEDIUM, or HIGH priority based on the total revenue amount.

### Consumer Processing

The Kafka consumer received enriched sales transaction messages from the Kafka topic.

The consumer processed up to 10 messages during testing and consumed each
message sequentially from the beginning of the topic.

For each message, the consumer:

- logged the message contents
- processed the Kafka event data
- appended the consumed records to an output CSV file

The consumer wrote the processed records to consumed_sales_kjleopold.csv

The consumed records included both the original dataset fields and the
additional enriched fields added by the producer, including total_revenue and
order_priority

The consumer did not filter out records, but it successfully processed and
stored the enriched streaming events generated by the producer.

### Experiments

Phase 4 Change

The producer configuration was modified to increase the number of streamed
messages from 3 to 10 by changing the PRODUCER_MESSAGE_COUNT setting in the .env
file.

The message interval was also changed from 2.0 seconds to 1.0 second using the
PRODUCER_MESSAGE_INTERVAL_SECONDS setting. This allowed the producer to simulate
a faster real-time streaming workflow and increased the volume of messages
moving through Kafka during testing.

In addition, a custom Kafka topic named streaming-02-kafka-kjleopold was created and used for the project.
